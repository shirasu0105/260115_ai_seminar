# ワイン品種予測AIモデル 精度向上検証レポート

## 1. 目的
本検証の目的は、**ワイン品種予測AIモデルの精度を、単発の高スコアではなく再現性を伴って改善すること**である。

そのために、以下の3ステップで検証を進めた。

1. **評価土台の構築**  
   単一条件の結果だけでなく、データ分割による性能ばらつきを確認し、安定した評価方法を整備する。
2. **分析**  
   精度が悪化するケースを特定し、どの品種間で誤分類が起きているかを明らかにする。
3. **改善**  
   原因分析を踏まえ、軽量なモデル調整で最悪ケースの下振れを抑えつつ、平均精度の改善を目指す。

---

## 2. 目次
1. 現行モデルのばらつき検証  
2. 最悪ケース（seed=384）の誤分類分析  
3. RandomForest軽量調整（class_weight）による改善  
4. 全体まとめと今後の展望  

---

## 3. 現行モデルのばらつき検証

### 3.1 課題
現行条件では accuracy=1.0000 を確認していたが、**単一seedの結果だけではモデル性能を正しく評価できていない可能性**があった。

### 3.2 実施内容
**データ分割seedを100通り変えて評価し、精度のばらつきを確認した。**

- `train_test_split` の `random_state` を100通り変更
- `RandomForestClassifier(random_state=40)` で毎回学習・評価
- 品種比率を保つため `stratify=y` を使用

前回との差分: [3a3b100..aef240e](../../compare/3a3b100..aef240e)

### 3.3 結果
**単発では1.0000だったが、実際には分割次第で accuracy が 0.9167 まで低下することが分かった。**

| 指標 | 値 |
|---|---:|
| Before（単発評価） | 1.0000 |
| After（100回平均） | 0.9831 |
| 標準偏差 | 0.0204 |
| 最小値 | 0.9167 |
| 最大値 | 1.0000 |
| 差分（平均 - Before） | -0.0169 |

### 3.4 所見
- 単発の accuracy=1.0000 は再現しうるが、**安定して得られる性能ではない**
- 評価値には分割条件によるばらつきがあり、**まず評価土台を整える必要がある**ことが確認できた

---

## 4. 最悪ケース（seed=384）の誤分類分析

### 4.1 課題
ばらつき検証の結果、最悪ケースは `seed=384` で accuracy=0.9167 だった。  
ただし、**どのクラス間で誤分類が起きているかは未確認**だった。

### 4.2 実施内容
**最悪ケースの混同行列と分類レポートを出力し、誤分類の内訳を分析した。**

- 100通りの評価フローは維持
- `split_seed==384` の場合のみ `confusion_matrix` と `classification_report` を表示
- 条件は `test_size=0.2, stratify=y, model_seed=40, runs=100` で固定

前回との差分: [aef240e..da4dd24](../../compare/aef240e..da4dd24)

### 4.3 結果
**誤分類は主に class0 と class1 の間で発生し、特に class1 の recall 低下が最悪ケースの主因だった。**

#### seed=384 の評価結果

| 指標 | 値 |
|---|---:|
| accuracy | 0.9167 |
| class0 precision / recall / f1 | 0.92 / 0.92 / 0.92 |
| class1 precision / recall / f1 | 0.92 / 0.86 / 0.89 |
| class2 precision / recall / f1 | 0.91 / 1.00 / 0.95 |

#### confusion matrix

| 実際＼予測 | class0 | class1 | class2 |
|---|---:|---:|---:|
| class0 | 11 | 1 | 0 |
| class1 | 1 | 12 | 1 |
| class2 | 0 | 0 | 10 |

### 4.4 所見
- **主な誤分類は class0 ↔ class1**
- 特に **class1 の recall=0.86** が最悪ケースの精度低下に効いている
- class2 は recall=1.00 であり、比較的安定して識別できている

### 4.5 仮説
- class0 と class1 は一部特徴量分布が重なっており、**分割サンプル次第で境界がぶれやすい**
- テストデータが36件と少ないため、**数件の取り違えでも accuracy が大きく変動する**

---

## 5. RandomForest軽量調整（class_weight）による改善

### 5.1 課題
誤分類分析により、最悪ケースでは class1 の recall 低下が主要因であることが分かった。  
そこで、**比較可能性を保ちながら軽微な変更で下振れを改善できるか**を確認した。

### 5.2 実施内容
**`class_weight='balanced'` を追加し、最悪ケースと平均精度の両方が改善するか検証した。**

- `RandomForestClassifier` に `class_weight='balanced'` を追加
- その他条件は固定  
  - `n_estimators=100`
  - `model_seed=40`
  - `runs=100`
  - `test_size=0.2`
  - `stratify=y`
- `seed=384` の詳細分析ロジックは維持

前回との差分: [da4dd24..5960dfd](../../compare/da4dd24..5960dfd)

### 5.3 結果
**軽量調整だけで、最悪ケース・平均精度・ばらつきのすべてが改善した。**

#### 全体比較

| 指標 | 調整前 | 調整後 | 差分 |
|---|---:|---:|---:|
| 最小 accuracy | 0.9167 | 0.9444 | +0.0278 |
| 平均 accuracy | 0.9831 | 0.9858 | +0.0027 |
| 標準偏差 | 0.0204 | 0.0178 | -0.0026 |

#### seed=384 の比較

| 指標 | 調整前 | 調整後 | 差分 |
|---|---:|---:|---:|
| accuracy | 0.9167 | 0.9444 | +0.0278 |
| class1 recall | 0.86 | 0.93 | +0.07 |

#### seed=384（調整後）の confusion matrix

| 実際＼予測 | class0 | class1 | class2 |
|---|---:|---:|---:|
| class0 | 11 | 1 | 0 |
| class1 | 0 | 13 | 1 |
| class2 | 0 | 0 | 10 |

### 5.4 所見
- `class_weight='balanced'` の追加だけで、**最悪ケースの落ち込みを抑制**できた
- 平均精度も微増し、ばらつきも縮小した
- class0↔class1 の取り違えが1件減少し、**class1 の recall 改善が効果として確認できた**

---

## 6. 全体まとめと今後の展望

### 6.1 全体まとめ
**単発で accuracy=1.0000 に見えていたモデルを、ばらつき込みで評価し直し、最悪ケースまで含めて改善できる状態に引き上げた。**

#### From → To
- **From:** 単発評価では高精度に見えるが、分割による性能ばらつきや誤分類傾向が見えていない状態
- **To:** ばらつき・誤分類要因を把握したうえで、軽量調整により**最悪ケース改善、平均精度向上、安定性向上**まで確認できた状態

### 6.2 本検証で得られた成果
- 単一seed評価の限界を明確化し、**100回評価による再現性ある評価土台**を構築できた
- 最悪ケースの誤分類傾向を分析し、**class0/class1 が主要な改善対象**であると特定できた
- `class_weight='balanced'` のみの軽量調整で、**最悪精度・平均精度・安定性を同時に改善**できた

### 6.3 今後の展望
今回の改善で一定の効果は確認できたが、まだ以下の課題が残る。

- **評価データ数が少ない**  
  test件数が36件と小さく、数件の誤分類で精度が大きく変動するため、より安定した評価には cross-validation などの導入が有効と考えられる。
- **class0 / class1 の境界分析が不足している**  
  特徴量分布や重要特徴量を可視化し、どの特徴が境界の不安定さに影響しているかを追加分析する余地がある。
- **改善施策の比較がまだ限定的**  
  今回は `class_weight` のみを試したため、今後はハイパーパラメータ調整（例: `max_depth`, `min_samples_leaf`）や他モデル比較も行うことで、さらなる改善が期待できる。
